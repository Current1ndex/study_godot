# Godot 关键概念

游戏可以看作一棵由<b>节点</b>构成的<b>树</b>，节点分组可以形成<b>场景</b>，节点之间通过<b>信号</b>进行通信。
1. 节点：最小的构筑块，Godot 提供了丰富的节点类型库。  
所有节点都有以下特性：名称、属性、回调+更新、可扩展、与其他节点进行嵌套。
2. 场景：场景或预制件(Prefab)，由一个或者多个节点组成，可以互相嵌套。
3. 场景树：游戏的所有场景构成的树。
4. 信号：事件发生后发出信号，无需在代码中硬连接节点就能实现互相通信。


内置文档查阅方法：<u>(Fn +) F1</u>。Godot 支持 4 种脚本语言，可以用 C 或 C++ 来实现高要求算法，用 GDScript 或 C# 来编写大部分游戏逻辑。


C# 开发注意：
1. 使用 C# 开发时要注意其垃圾回收机制。
2. 截止到 2026.04.19，Godot 4 中用 C# 编写的项目无法导出到 Web 平台。
